So You Want To Make A Website
  First off, reconsider. I really don't recommend creating a website from scratch—just use a website builder. Unless you have:
 1. previous experience,
 2. a lot of free time,
 3. a need for a custom solution,
a run-of-the-mill website builder will cover 95% of your needs. This is coming from someone who met none of the above three requirements. However, if you're still undeterred, the first thing you should determine is how complicated your website will be.

  This website is very, very simple. It is essentially just a couple of fancy static pages—no need to request anything or load from an external database. This is the first consideration: is your website or ? Static websites are far and away easier to create, and so this post will on focus on them.

  Because this is a static website, I did not implement any JS framework. Rather, this is built using Tailwindcss—a CSS framework. I recommend this framework due to its high flexibility. It's essentially just CSS shorthand; rather than write:


    .link {
        text-decoration-line: underline;
        text-decoration-color: #0ea5e9;
        font-weight: 600;
    }

    .link:hover {
        text-decoration-color: #7dd3fc;
    }


    <div class="link">Website Link</div>

Tailwind will automatically generate the relevant CSS for you (the majority of CSS styles and even custom extensibility is supported), which allows the above snippet to be replaced with:


    <div class="underline decoration-sky-500 font-semibold hover:decoration-sky-300">
        Website Link
    </div>

  Admittedly, this can lead to quite , but I've found the increase in productivity and quicker iteration time to be worth it. One thing to note is that unlike some other frameworks, Tailwindcss does not come with pre-made assets out of the box. You will either have to use another library (i.e. Tailwind Elements) or create them yourself.

  On the other hand, frameworks such as Bootstrap will provide a lot of default functionality which streamlines the website creation process a lot. However, this causes the resultant websites to often look homogenous and a bit bland. If you are just trying to get a website up quickly, I would recommend using Bootstrap. But if you want a more customized solution like this website, Tailwindcss is a great pick.


Templates/Components
  Websites will commonly have repeated elements. For example, our about page has a very similar ribbon pattern for each person. Initially, I hard coded just about everything, but every time I wanted to make a small change I would have to manually propagate that change to every instance. Even worse, each team would have a different color scheme, necessitating even more editing.

  Instead, consider using some form of templates. In place of manually copy pasting, I've used a small JS script to automatically do that and replace variables as needed. The actual HTML element is then replaced with my own custom element:


    <div 
        class="about-member"
        about-right
        about-title="LEO WANG"
        about-src="../docs/images/headshots/leo.jpg"
        about-color="red"
        about-desc="I'm passionate about simulations and enjoy designing robots and strategies around new games each year. I have been resigned to Website Team :(">
    </div>

Which I then parse with some custom JS code. It's really simple; it finds each relevant HTML element, then inserts the custom HTML with any needed custom values.


    const aboutInit = () => {
        const am = document.querySelectorAll(".about-member"); // find each custom element
        am.forEach((ele) => {
            // get the metadata
            const n = ele.getAttribute('about-title');
            const d = ele.getAttribute('about-desc');
            const s = ele.getAttribute('about-src');
            const c = ele.getAttribute('about-color');

            let dir = 'left';
            if (ele.hasAttribute('about-right')) dir = 'right';
            else if (ele.hasAttribute('about-middle')) dir = 'middle';

            // apply the custom html
            ele.innerHTML += memberDir(n, d, s, colorPreset[c], direction[dir], memberId);
            memberId++;
        });
    };

    // function that returns the html and applies custom values
    // only the elements that differ per ribbon are shown
    const memberDir = (name, desc, src, color, dir, id) => {
        return `

            <div class="${dir['sort']} ...">
                <div class="...">
                    <div class="...">
                        <div class="${dir['origin']} ${dir['margin']} ${dir['gradient']} ${color['from']} ${color['via']} ..."></div>
                    </div>
                    <div class="${dir['sort']} ...">
                        <img src="${src}" class="${color['ring']} ..."/>
                        <p id="about-id-${id}" class="...">${name}</p>
                    </div>
                </div>
                <p class="...">${desc}</p>
            </div>

        `;
    };

  It's not a perfect fix, but it definitely helps clean up the code a lot especially if you use a framework that has built-in support for components. Caches
  Most modern browsers will implement some form of a cache. Similar to cookies, these caches store some data about the website. However, whereas cookies usually store some user metadata, a cache is used to store information about the website itself. This is usually in the form of images or videos, with the idea being to cache larger images/videos so that the user would not need to download them if they return to the website. Instead, the content would already be available on the user's browser.

  Although this is extremely helpful in production, this can cause a few bugs during development. The browser will only request an updated image if it detects some part of the website has changed. For simple static websites such as this page, that will usually be through updating the HTML source code. Notably, this means that the browser will not request an update if something that the HTML code references changes.

  So, if your source code references a picture, but then you change that picture on the backend—without changing the source code—the page will never show the updated picture unless you clear your cache or change the HTML.

  For example, our about page features an image carousel. For standardization purposes, each contained image is named as 1.jpg, 2.jpg, 3.jpg ... n.jpg, where the number corresponds to which position they occupy within the carousel. If, for example, I wanted to switch 1.jpg with a new image, it should be as simple as deleting the old image, inserting a new one, and renaming it to 1.jpg right? However, since the HTML code never changed, the browser will still load the old, cached image. To have the new image show, you have to force the browser to update, usually through changing the HTML code or clearing your cache.
